JWhisper Relay Server Overview

The jw-relay artifact is a lightweight, zero-knowledge switchboard server responsible for managing user identities and routing end-to-end encrypted envelopes. The Relay serves strictly as an untrusted courier — it exposes public keys but is cryptographically restricted from inspecting message text payloads.

Key Structural Pillars

  • io.github.artshp.jwhisper.relay.storage: Manages data persistence using in-memory hash table. Users need to register after every server's restart.
  • io.github.artshp.jwhisper.relay.network: Orchestrates active sessions via thread-safe routing tables and implements concurrent Virtual Thread task dispatchers capable of managing thousands of simultaneous node connections cleanly.
  • io.github.artshp.jwhisper.relay.log: Integrates advanced Mapped Diagnostic Context (MDC) trackers paired with a custom Logback layout converter to contextually color-code system telemetry from user-driven streams.
